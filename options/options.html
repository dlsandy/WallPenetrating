使用说明

Chrome 扩展通过 PAC 脚本将指定域名路由到本地 SOCKS5 代理

导入与导出

  • 网址规则与节点均支持 JSON 文件导入/导出（选项页）
  • 节点密码仅保存在本机，不会同步到 Google 账号
  • 导出文件含敏感信息，请妥善保管

快速开始

  1. 完成运行环境中的一次性 Native Host 配置（双击「一键安装.cmd」）
  2. 在「节点管理」中添加 AnyTLS 节点（支持 URI 导入）
  3. 在「网址管理」中添加需要代理的域名
  4. 开启「启用分流」

节点订阅

  • 在「节点管理」填写订阅链接，点击「刷新订阅」
  • 支持每行一个 anytls:// 链接、Base64 编码或 JSON 数组
  • 可开启定时自动刷新（后台替换全部节点）
  • 订阅设置会随「导出全部配置」一并备份
  • 通过 Native Host 拉取，无需额外网络权限

URI 格式示例

anytls://password@example.com:443/?sni=real.example.com
anytls://password@1.2.3.4:8443/?insecure=1

多节点分流

  • 每条代理规则可指定独立节点；未指定则使用「默认节点」（节点管理中「设为默认」）
  • 不同规则可走不同 AnyTLS 节点，sing-box 会同时监听各节点的本地 SOCKS 端口
  • 全局代理模式下仍只使用默认节点

v1.10 新增

  • 规则外 404 自动走代理：未在规则内的站点直连返回 404 时，自动加入临时代理规则并重载（弹窗 / 侧边栏开关，默认关）
  • 临时代理规则：会话内有效，关闭浏览器后清空；可在「网址管理 → 临时代理规则」转为永久规则
  • 弹窗与选项页显示扩展版本号；侧边栏可设置项目网址

匹配类型说明

  • 域名：匹配 example.com 及其子域名
  • 通配符：如 *.example.com
  • 正则：对 host 进行正则匹配（高级）
